수학적 명료함을 유지하면서, 각 수식과 개념이 어떤 의미를 가지는지 연결 고리(맥락)를 조금 더 보강하여 재작성했습니다.

### **1. 일반화 및 표본 복잡도 (Generalization & Sample Complexity)**
기계학습의 핵심은 학습 데이터에서의 성능이 실제 미지의 데이터에 대한 성능으로 이어지는지 보장하는 것입니다.

*   **오차의 정의 (이진 분류, $y \in \{0, 1\}$ 가정)**:
    *   **경험적 위험 (Empirical Risk):** 우리가 실제로 관측하고 최소화할 수 있는 학습 데이터 상의 오분류 비율입니다.
        $$\hat{\epsilon}(h) = \frac{1}{n}\sum 1\{h(x^{(i)}) \ne y^{(i)}\}$$
    *   **일반화 오차 (Generalization Error):** 모델이 실제 데이터 분포 $\mathcal{D}$에서 틀릴 진짜 확률입니다.
        $$\epsilon(h) = P_{(x,y)\sim \mathcal{D}}(h(x) \ne y)$$
    *   **경험적 위험 최소화 (ERM):** 학습 오차를 가장 줄이는 가설을 찾는 과정입니다. $\hat{h} = \arg\min_{h \in \mathbb{H}} \hat{\epsilon}(h)$.

*   **균등 수렴 및 오차 상한 (Union Bound & Chernoff Bound)**:
    *   유한한 크기($|\mathbb{H}| = k$)를 가진 가설 공간에서, 어떤 가설이라도 학습 오차와 실제 오차의 차이가 $\gamma$보다 클 확률은 지수적으로 억제됩니다.
        $$P(\exists h \in \mathbb{H}, |\epsilon(h) - \hat{\epsilon}(h)| > \gamma) \le 2k e^{-2\gamma^2 n}$$
    *   확률 $1-\delta$ 이상에서 다음 상한선이 보장됩니다.
        $$\epsilon(\hat{h}) \le \min_{h \in \mathbb{H}} \epsilon(h) + 2\sqrt{\frac{1}{2n}\log\frac{2k}{\delta}}$$

*   **편향-분산 상충 (Bias-Variance Tradeoff)**:
    *   가설 공간의 크기 $k$가 커질수록 더 복잡한 모델을 표현할 수 있어 자체적인 한계 오차($\min \epsilon(h)$)는 작아집니다(편향 감소). 
    *   하지만 오차 상한선의 페널티 항인 $2\sqrt{\frac{1}{2n}\log\frac{2k}{\delta}}$가 커져 오차의 변동성이 증가합니다(분산 증가). 
    *   특정 오차 범위 $\gamma$를 보장하기 위한 필요 표본 수(Sample Complexity)는 $n \ge \mathcal{O}\left(\frac{1}{\gamma^2}\log\frac{k}{\delta}\right)$입니다.

### **2. Vapnik-Chervonenkis (VC) 차원**
실제 기계학습 모델은 파라미터가 연속적이므로 가설 공간의 크기 $k$가 무한대입니다. 이를 해결하기 위해 모델의 수용력(Capacity)을 측정하는 개념이 도입되었습니다.

*   **정의**: 가설 공간 $\mathbb{H}$가 모든 가능한 라벨 조합을 완벽히 분류(Shatter)할 수 있는 최대 데이터 포인트의 수를 $VC(\mathbb{H})$라고 합니다. 
    *   예컨대 2차원 선형 분류기는 최대 3개의 점을 모든 경우의 수로 분류할 수 있으므로 $VC(\mathbb{H}) = 3$입니다.
*   **VC 오차 상한 ($D = VC(\mathbb{H})$)**:
    *   확률 $1-\delta$ 이상에서 다음 부등식이 성립합니다.
        $$\epsilon(\hat{h}) \le \epsilon(h^*) + \mathcal{O}\left(\sqrt{\frac{D}{n}\log\frac{n}{D} + \frac{1}{n}\log\frac{1}{\delta}}\right)$$
    *   **표본 복잡도:** 최적 오차에 근접하기 위해 필요한 학습 데이터 수 $n$은 모델의 파라미터 수(주로 VC 차원에 비례)에 대략 선형적으로 비례($n = \mathcal{O}_{\gamma, \delta}(D)$)합니다.

### **3. 정규화 (Regularization)**
파라미터의 크기나 0이 아닌 개수 등을 모델의 복잡도 척도로 삼아, 이에 페널티를 부여함으로써 과적합(Overfitting)을 방지합니다.

*   **목적 함수**: 기존 손실 함수 $J(\theta)$에 복잡도 항 $R(\theta)$를 추가하여 최적화합니다.
    $$J_\lambda(\theta) = J(\theta) + \lambda R(\theta) \quad (\lambda \ge 0)$$
    *   $\lambda$가 0에 가까우면 원본 데이터 적합에 집중하고, 매우 크면 원래의 손실 함수가 제 역할을 하지 못합니다.

*   **정규화 항 $R(\theta)$의 종류**:
    *   **$L_2$ Norm (가중치 감쇠, Weight Decay):** $R(\theta) = \frac{1}{2}\|\theta\|_2^2$. 최적화 시 매 단계마다 가중치를 일정 비율($(1-\eta\lambda)$)로 감소시켜 파라미터가 너무 커지는 것을 막습니다.
        $$\theta \leftarrow (1-\eta\lambda)\theta - \eta\nabla J(\theta)$$
    *   **$L_1$ Norm (LASSO):** 미분 불가능한 $L_0$ Norm(0이 아닌 파라미터 개수)을 다루기 쉽게 완화한 형태로, 모델 파라미터의 희소성(Sparsity)을 유도하여 탐색 공간을 좁힙니다. $R(\theta) = \|\theta\|_1$.

*   **암묵적 정규화 (Implicit Regularization)**:
    *   딥러닝은 학습 오차가 0인 전역 최소점(Global Minima)이 무수히 많지만, 어떤 최적화 방식을 쓰느냐에 따라 도달하는 최소점이 달라져 일반화 성능이 극적으로 변합니다.
    *   명시적인 수식 항이 없어도, 큰 학습률(Learning Rate), 작은 가중치 초기화, 작은 배치 크기(Batch Size), 모멘텀(Momentum) 등의 하이퍼파라미터 선택 자체가 과적합을 막는 암묵적 정규화 역할을 합니다.

### **4. 교차 검증 (Cross Validation)**
주어진 학습 데이터에서 오차가 가장 작은 모델만 고르는 방식은, 데이터에 과하게 맞춰진 고차원 모델(High-variance)을 선택하는 위험을 초래합니다. 이를 막기 위해 데이터를 나누어 모델을 평가합니다.

*   **Hold-out CV**: 전체 데이터 $S$를 학습용 $S_{train}$과 검증용 $S_{CV}$ (약 30%)로 나누어, $S_{CV}$에서의 오차 $\hat{\epsilon}_{S_{CV}}(h)$가 가장 작은 모델을 최종 선택합니다. 단점은 전체 데이터의 일부를 학습에 활용하지 못해 낭비가 생긴다는 점입니다.
*   **$k$-Fold CV**: 데이터가 부족할 때 유용합니다. $S$를 $k$개로 분할하여 1개를 검증용으로, 나머지 $k-1$개를 학습용으로 쓰는 과정을 $k$번 반복해 평균 테스트 오차를 구합니다.
    *   극단적으로 조각 수 $k$를 전체 데이터 개수 $m$과 동일하게 설정하는 방식을 Leave-one-out CV라고 합니다.

### **5. 빈도주의 vs 베이즈 통계 (Frequentist vs Bayesian)**
통계학적 관점에 따라 모델 파라미터 $\theta$를 추정하는 철학이 달라지며, 이는 정규화 효과로 직결됩니다.

*   **빈도주의 (Frequentist)**:
    *   $\theta$를 알 수는 없지만 '고정된 상수'로 취급합니다.
    *   **최대 우도 추정 (MLE):** 관측된 데이터가 나올 확률을 가장 크게 만드는 파라미터를 점 추정합니다.
        $$\theta_{MLE} = \arg\max_\theta \prod_{i=1}^n p(y^{(i)}|x^{(i)}; \theta)$$

*   **베이즈 관점 (Bayesian)**:
    *   $\theta$를 '확률 변수'로 취급하며, 데이터를 보기 전 우리가 가진 사전 믿음인 사전 분포 $p(\theta)$를 사용합니다.
    *   데이터를 관측한 후, 베이즈 정리를 통해 사후 분포(Posterior)를 업데이트합니다.
        $$p(\theta|S) = \frac{\left(\prod p(y^{(i)}|x^{(i)}, \theta)\right)p(\theta)}{\int_\theta \left(\prod p(y^{(i)}|x^{(i)}, \theta)\right)p(\theta)d\theta}$$
    *   이론적으로 가장 정확한 방식인 완전 베이즈 예측(Fully Bayesian prediction)은 모든 $\theta$ 확률공간에 대해 적분($\int_\theta p(y|x,\theta)p(\theta|S)d\theta$)해야 하므로 계산이 매우 어렵습니다.

*   **최대 사후 확률 (MAP)**:
    *   완전 베이즈 적분 대신, 사후 확률 자체를 가장 높이는 단일 파라미터 $\theta$를 찾는 근사 기법입니다.
        $$\theta_{MAP} = \arg\max_\theta \prod_{i=1}^n p(y^{(i)}|x^{(i)}, \theta)p(\theta)$$
    *   사전 분포 $p(\theta)$를 정규 분포 $\mathcal{N}(0, \tau^2 I)$로 가정하면, 수학적으로 가중치 크기를 제한하는 효과가 발생합니다. 이로 인해 $\|\theta_{MAP}\| < \|\theta_{MLE}\|$ 형태가 되어 MLE보다 과적합에 훨씬 강합니다.
